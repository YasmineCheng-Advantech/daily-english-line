# 越南文單字生成 Prompt 範本

跟英文那份（`PROMPT_TEMPLATE.md`）同一套流程：貼給任何 LLM（ChatGPT、Gemini、Claude.ai 等），請它輸出 JSON，存成一個新檔案（例如 `vi_batch.json`），再用 `scripts/merge_words.py --lang vi` 合併進 `data/vi/words.json`。

不需要在 prompt 裡貼現有單字清單當排除名單——重複的字交給 `merge_words.py` 自動比對 `text` 欄位過濾掉即可。

> **音系採北部（河內）腔。** 專案原本規劃南部音，但當初的規格把「r 發 z 音」寫成南部特徵，實際上那是北部音；南部音的 `r` 保留捲舌、近英文 r，只有 `d`／`gi` 才發 y 音。兩者不能混，所以整批統一成北部音。如果之後要做南部音版本，另外開一批 `--region south` 的資料，不要混進同一個檔案。

---

## Prompt 內容（複製貼上，`{N}` 和 `{TOPICS}` 換成你要的數量與主題）

```
請生成 {N} 個實用的越南文單字/片語，主題涵蓋：{TOPICS}
（例如：數字, 時間, 點餐, 問路, 購物... 或直接說「不限主題，涵蓋日常生活情境」）

學習者是台灣人，母語繁體中文，程度是初學到初中階。請以「能在越南日常生活實際用出來」為選字標準，
避免過於書面或罕用的詞。

【發音規則：一律採北部（河內）腔】
羅馬拼音請用「英文式直覺近似拼音」，不要用 IPA 音標。北部音要點：
- d、gi、r 三者都發「z」音（dạ → za、gì → zi、rất → zut）
- 但 đ（有橫槓的 d）不受此規則影響，仍是 d 音（được → duoc）
- -nh 與 -ng 要分清楚（anh → anh，不是 ang）
- không 收清楚的 -ng 鼻音（khong），嘴唇不閉起來
- kh- 是喉部摩擦音（khỏe → khwe）
- ơ 近「u」的短音（cảm ơn → kam un）

輸出一個 JSON 陣列，每個物件必須恰好包含這 10 個欄位：

- "type": "word" / "phrase" / "dialogue" 三選一
- "topic": 主題分類，用繁體中文（例如 "數字"、"點餐"、"問路"）。
  優先沿用現有主題，同一個概念不要一直發明新名稱。現有主題：打招呼, 自我介紹
- "text": 越南文原文，**必須帶完整聲調符號**（例如 "xin chào" 不可寫成 "xin chao"）
- "meaning_zh": 繁體中文意思，簡短清楚。若有使用情境限制請在括號註明
  （例如 "你好（較正式）"、"是的（禮貌回應；北部常用 vâng）"）
- "romanization": 這個字的英文式近似拼音（依上面北部音規則）
- "tone_note": 聲調或發音容易跑掉的提醒，繁體中文，一句話。沒有特別要提醒的就給空字串 ""
- "example": 一句越南文例句，帶完整聲調符號，長度控制在 10 個字以內
- "example_zh": 例句的繁體中文翻譯
- "example_rom": 例句的英文式近似拼音。可給空字串 ""
- "grammar_point": 隨這個字帶出的文法點，繁體中文，一句話講完，要實用
  （例如 "「形容詞 + không?」= 是不是...？最常用的疑問句型"）。沒有的話給空字串 ""

請確保：
1. 越南文的聲調符號完全正確——這是最重要的一點，寧可少給幾個字也不要拼錯
2. 這批資料裡面沒有重複的 "text"
3. 稱謂系統要正確：越南文的「你/我」隨對方年齡性別改變
   （anh 哥 / chị 姐 / em 弟妹 / bạn 同輩朋友），例句用到時請確保搭配合理
4. 每個物件都恰好有這 10 個 key，不多不少
5. 不要自己編 "id"、"lang"、"region" 這三個欄位，合併腳本會自動補
6. 直接輸出純 JSON 陣列，不要加任何說明文字或 markdown 的 ``` 標記
```

## 拿到結果之後

1. 把 LLM 回傳的 JSON 存成檔案，例如 `vi_batch.json`
2. 執行：
   ```bash
   python3 scripts/merge_words.py --lang vi vi_batch.json
   ```
3. 腳本會自動：
   - 驗證格式（10 個 key、`type` 是否合法、必填欄位是否為非空字串）
   - 跟現有 `data/vi/words.json` 比對 `text`（不分大小寫），過濾掉重複的字
   - **自動補上 `id`（接續 vi-0011、vi-0012…）、`lang: "vi"`、`region: "north"`**
   - 印出「新增幾個、跳過幾個重複、跳過幾個格式錯誤」的統計
   - 把結果寫回 `data/vi/words.json`

   要做南部音版本的話加 `--region south`。

4. 檢查統計數字合理後，先預覽排版：
   ```bash
   python3 push.py --lang vi --dry-run
   ```
   確認沒問題再 `git add data/vi/words.json && git commit -m "..." && git push`

## 建議的擴充節奏

`data/vi/words.json` 目前只有 10 筆，分成「打招呼」「自我介紹」兩個主題各 5 筆。而 `daily-vi.yml` 每晚 21:00 推 5 筆，**所以第 3 天起就會開始重複第一輪的內容**（`source` 會標成 `bank-recycled`）。

建議先累積到 **50 筆以上**（約 10 天份）再讓排程長期跑。一次生成 30-50 筆品質最穩定，越南文比英文更容易出錯（聲調符號、稱謂搭配），一次要太多容易開始拼錯或亂湊。

主題擴充建議順序（由高頻到低頻）：

| 順序 | 主題 | 說明 |
|---|---|---|
| 1 | 數字、時間 | 幾乎所有情境都會用到 |
| 2 | 點餐、購物 | 出差／旅遊最高頻 |
| 3 | 問路、交通 | 同上 |
| 4 | 禮貌用語、道歉道謝 | 建立好感度 |
| 5 | 辦公室、會議 | 銜接你的商用英文情境 |
| 6 | 數量詞、方位 | 文法基礎 |

每次生成後務必人工抽查幾筆的聲調符號——LLM 生越南文最常見的錯就是漏掉或標錯聲調，而錯的聲調在越南文裡會直接變成另一個字。
